# Saikiran Financial Portfolio

A sleek, responsive, GitHub Pages-ready portfolio website for Saikiran, focused on financial education, LLQP licensing, WFG business mentorship, life insurance, wealth protection, and entrepreneurship.

## Files

Upload these files/folders to the root of the GitHub repository:

```text
index.html
styles.css
script.js
README.md
assets/
```

## How to add Saikiran's profile picture

1. Rename the image to one of these exact names:
   - `saikiran.jpeg` preferred
   - `saikiran.jpg`
   - `saikiran.png`
2. Put the image inside the `assets` folder.
3. Final path should look like this:

```text
saikiran.jpeg
```

The website automatically checks these image names in this order:

```text
saikiran.jpeg
saikiran.jpg
saikiran.png
```

If no image is found, the site shows the fallback initials `SG`.

## How to upload files to GitHub

1. Open the GitHub repository.
2. Click **Add file**.
3. Click **Upload files**.
4. Drag and drop all website files and the `assets` folder.
5. Click **Commit changes**.

To upload only the profile picture:

1. Open the repository.
2. Click the `assets` folder.
3. Click **Add file**.
4. Click **Upload files**.
5. Upload `saikiran.jpeg`.
6. Click **Commit changes**.

## How to publish on GitHub Pages

1. Go to repository **Settings**.
2. Click **Pages**.
3. Under **Build and deployment**, choose **Deploy from a branch**.
4. Choose branch: `main`.
5. Choose folder: `/root`.
6. Click **Save**.

## Notes before publishing

Please verify all numbers and claims before publishing, including:

- `1500+ families served by team`
- `$1,000,000+ team earnings`
- `500+ families served in 12 months`
- `50+ individuals mentored`
- `4+ years in financial industry`
- Business model comparison numbers such as startup costs, revenue estimates, profit potential, and licensing costs

Financial, insurance, and business opportunity websites should avoid implying guaranteed results. This site uses education-focused and compliance-friendly language, but Saikiran should still review it for accuracy, trademark usage, income-claim wording, and regulatory/compliance approval before publishing.
